# 短篇随笔作品集网站——设计文档

**日期：** 2026-06-14  
**状态：** 待审核

---

## 一、项目概述

为短篇随笔作家搭建个人作品集网站。纯文字中文随笔，当前约 10 篇，按主题分类，持续更新。追求极简与个性的融合——"禅意水墨"美学。

## 二、用户画像与使用场景

- **作者本人**：通过 Markdown 文件管理随笔，Git 推送即发布
- **读者**：浏览分类作品、阅读正文、了解作者、通过搜索找到内容、通过联系方式触达作者

## 三、功能范围

### 包含
- 首页：作者名 + 站点名 + 一句话介绍 + 主题分类入口 + 最新随笔列表
- 作品列表页：按主题分组展示全部随笔，点击标签筛选
- 随笔正文页：传统书版排版，分类标签 + 日期 + 上下篇导航
- 关于我页面：独立页面，醒目可访问，完整自我介绍（Markdown）
- 联系方式页面：邮箱 + 社交链接
- 搜索页面：前端静态全文搜索，即时响应
- 暗色模式：不作为本期需求

### 不包含（本期）
- 评论系统（无后端预算）
- 图片/多媒体（纯文字随笔）
- 英文/多语言（中文写作）
- RSS 订阅
- 后台管理界面

## 四、视觉设计

### 4.1 风格定位
**禅意水墨**——留白如宣纸，衬线字如墨，以一两笔赭石色破白。气质安静、克制、有东方韵味，兼具现代极简的功能清晰和传统书卷的美感。

### 4.2 配色
| 角色 | 色值 | 用途 |
|------|------|------|
| 页面背景 | `#faf9f6` | 米白，宣纸质感 |
| 主体文字 | `#2d2c28` | 深棕，柔和于纯黑 |
| 点缀色 | `#c4a882` | 赭石金，装饰线和强调 |
| 细线/分隔 | `#e8e3db` | 暖灰，低调分隔 |
| 次级文字 | `#888` ~ `#aaa` | 标签、日期、导航 |

### 4.3 字体
- **正文/标题：** 思源宋体（Noto Serif SC），Google 开源免费商用
- **导航/标签/元信息：** 系统黑体栈（`system-ui, -apple-system, sans-serif`）
- **回退字体：** Georgia, serif（英文部分）

### 4.4 正文页排版（核心阅读体验）
```
字号：14px（桌面）/ 16px（移动端）
行高：2.2
段落样式：首行缩进两字（text-indent: 2em），段间不留空
文字颜色：#3a3a3a
最大阅读宽度：680px，居中
```

此方案在「舒朗」「紧凑」之间取中，采用中文传统书版的首行缩进范式，阅读沉浸感强，一屏承载信息适中。

### 4.5 间距系统
- 页面级左右边距：桌面 48px，移动端 20px
- 模块间垂直间距：以 24px 为基数的倍数（24/48/72px）
- 装饰细线与相邻内容间距：16-20px

## 五、信息架构

```

/essays            作品列表（按主题分组）
/essays/[theme]    主题下作品列表（可选）
/essays/[theme]/[slug]  随笔正文
/about             关于我（醒目独立页面）

### 导航
- 桌面端：顶部或侧边固定导航，包含「作品」「关于」「联系」「搜索」
- 移动端：汉堡菜单，展开后同桌面导航项

### 首页布局

│  [导航：作品 | 关于 | 联系 | 搜索]  │

### 作品列表页
- 按主题分组，每组有小标题
- 每篇显示：标题 + 日期 + 一句话摘要
- 支持点击主题标签筛选（前端即时过滤）

### 随笔正文页
```
┌─────────────────────────────┐
│  主题标签 · 写作日期          │
│                             │
│  标题（大号宋体）             │

### 关于我页面
- 独立导航入口
- Markdown 渲染，支持多段落 + 列表
- 排版同正文页但可稍宽（720px）

### 联系方式页面
- 邮箱（可点击）
- 可选社交链接：GitHub / Twitter / 豆瓣 / 即刻 等
- 简洁列表式，图标 + 文字

### 搜索页面
- 搜索框 + 结果列表
- 前端静态搜索（Fuse.js 或 lunr.js）
- 匹配标题、主题、正文摘要
- 即时过滤，无需回车

## 六、技术架构

### 技术栈
| 层面 | 选型 | 理由 |
|------|------|------|
| 框架 | Next.js (App Router) | 静态生成 + MDX 原生支持 + Vercel 一键部署 |
| 内容 | MDX (Markdown + JSX) | Markdown 写作，支持 frontmatter 元数据 |
| 样式 | Tailwind CSS | 精确控制排版和间距，契合禅意设计 |
| 搜索 | Fuse.js | 前端模糊搜索，零后端依赖 |
| 托管 | Vercel | 免费层够用，自动部署，全球 CDN |
| 版本控制 | Git + GitHub | 免费，管理内容变更历史 |

### 内容管理流程
1. 作者在 `/content/essays/[主题]/` 下新建 `.mdx` 文件
2. 文件顶部 frontmatter：`title` `date` `theme` `summary`
3. 正文使用 Markdown 写作
4. `git add` → `git commit` → `git push`
5. Vercel 自动检测推送，自动构建部署

## 七、设计原则

1. **文字是第一主角**——所有设计服务于阅读体验，不争夺注意力
2. **留白即语言**——沉默的空间和文字同等重要
3. **经典中文字体排版**——首行缩进、宋体正文字号、宽行距
4. **一抹赭石色点睛**——整体克制，用一处暖色制造记忆点
5. **零摩擦更新**——作者只需写 Markdown + git push
6. **零费用运维**——静态生成 + 免费托管，无服务器成本
